## Key Principles

This workspace enforces five principles:

1. **If it's not on disk, it didn't happen.** Context windows compress. Sessions
   end. The only thing that persists is files. Flush work every 5 operations.

2. **Approval-first, always.** Plan work, get approval, then execute. No
   speculative actions.

3. **Quality is a structural gate.** Quality checks happen during work, not after.
   See `.claude/rules/quality-gates.md` for the full procedure.

4. **Disagreements reveal ambiguity.** When agents disagree, the rules are
   ambiguous. Escalate and clarify — don't suppress.

5. **One decision at a time.** Binary decisions made sequentially compound into
   reliable outcomes.

## Session Handoff

This workspace uses session handoff checklists. At the end of every session:

1. Save all work to SSOT files
2. Sync derived files
3. Check SSOT integrity
4. Update progress tracker
5. Write next-session plan
6. Run QC on session work

This ensures no context is lost between sessions.

## Spec-Driven Development

This workspace enforces a spec-first workflow. Before any implementation:

1. **Type `/spec`** to start a new spec interactively — Claude will guide you
2. Or copy `specs/_template.md` manually, fill it in, and get approval
3. Only implement after the spec is approved

Your specs live in `specs/`. Each approved spec is the contract between you and
Claude — implementation follows the spec, not the conversation.

See `.claude/rules/sdd.md` for the full rules and exemptions.
